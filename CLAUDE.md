# CLAUDE.md

## What This Is

Declarative macOS system configuration using **nix-darwin** + **home-manager**, managed as a Nix flake. Configures system packages, Homebrew casks/brews, shell (fish), dev tools, and app settings for multiple Mac hosts.

## Key Commands

```bash
# Update all flake inputs
nix flake update

# Validate the flake
nix flake check

# Apply nix-darwin configuration (macOS)
nix run nix-darwin -- switch --flake '.#jacobcolvin@Jacobs-Mac-mini'

# Apply standalone home-manager configuration (Linux)
home-manager switch --flake '.#jacobcolvin@linux'
```

Host names use the format `username@hostname` — see `darwinConfigurations` and `homeConfigurations` in `flake.nix` for all available targets.

## Architecture

- **`flake.nix`**: Entry point. Two helpers: `mkDarwin` (nix-darwin + home-manager) and `mkHome` (standalone home-manager for Linux). Each host passes a `hostConfig` attrset.
- **`hosts/mac.nix`**: System-level nix-darwin config (Nix settings, Homebrew, user accounts). Homebrew lists compose base packages + `hostConfig.homebrew.extra*`.
- **`home/`**: Home-manager modules imported from `home/default.nix`. Each `.nix` file is a self-contained module for a tool domain (shell, git, editors, k8s, etc.).
- **`configs/`**: Raw config files, normally symlinked into `~/.config/` via `xdg.configFile` and `home.file`.

## Nix Conventions

- Modules use the standard pattern: `{ pkgs, hostConfig, ... }: { ... }`
- Packages use `with pkgs;` lists; host-specific extras use `map (name: pkgs.${name}) hostConfig.extraFoo`
- App configs are raw files in `configs/` linked via home-manager — not generated by Nix
- Homebrew is used for GUI casks and packages not in nixpkgs; managed declaratively with `cleanup = "zap"`
